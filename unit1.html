1.7
GDPR is an acronym for General Data Protection Regulation, created by the EU to protect the personal data of anyone considered to be a data subject, (a natural person), and to also keep up to date with the rapid global technological advances, its aim is to adjust the legal regulations regarding the processing and protection of personal data, including identifiers such as a name, identification number, location, or other such factors that specific to physical, physiological, genetic, mental/cognitive, economic, cultural and societal identities, which in current times, there is unprecedented access to and unscrutinized use of personal data by all types of businesses, one of its end goals is to allow data subjects (individuals that use any online service), a significant degree of control over the processing of their data, giving independent public authorities flexible supervisory powers and rights to punish any non compliance via fine or economic sanction.

In short, it regulates how companies or businesses store data, and so data flows must be designed with this regulation in mind, therefore must only collect data relevant for their current means, a data minimisation principle, instead of the common method of data retention of collecting and storing great swathes of data because it may be useful to use later.

Non compliance with these regulations will incur huge, economically crippling punishments, which will also harm a company's image, not only that, but in cases of Information Technology centred companies, that reside within its member states or outside of the EU, it is required that they submit to an independent authority, such as a data controller or processor who may be forced to take specific actions to remain compliant to the regulation within specific timelines that can usually conflict with business interests, such as temporary limiting on data processing or its outright banning, since its enforcement all companies wishing to do business within the EU/EEA have had to audit personal data flows and define a legal basis for their collection and retention, creating further obligations and expenses on the part of the company.

In the creation of my website I have used a good range of tools, such as Visual Studio code, trello, google fonts, etc, in this section I will state the advantages and disadvantages of each and conclude in their usefulness in regards to my approach to website creation.


Visual studio code - visual studio code is a cross platform source code editor mainly used for front end development, used to edit, view, run and debug the source code for applications, its advantages include its “intellisense” feature which is extremely helpful with individual productivity, including a code editing feature for code completion, parameter information, quick information, content assisting and code hinting, on all these features work in synchronicity to help the user complete new projects as quickly as their own individual skills can allow, continuously pointing out errors before debugging.


Another advantage would be its built in version control via github, when connected to github visual studio code makes it remarkably easy to see which files have been modified, unsaved and unpushed. Through the use of the inbuilt terminal, we can initialise git and perform various git commands, such as push, pull, rebase, publish, etc.


Another advantage would be its debugging feature and support, which help accelerate editing, compiling and other such things.


Trello - trello is a cross platform web application, which allows the user to send and receive files from their team, understand instantly when deadlines loom, through the use of cards, that can have due dates added to them, as the date nears the cards in question will change colour from white to yellow, and when the due date has passed it will change again to the colour red, but this can be avoided by marking the due date as complete. This in immensely helpful in regards to my own productivity, this is because, using this particular system pushes me to stay focused on one task at a time until it is, with certainty, completed.


The guiding principle behind trello is reductionism, to organise large projects by breaking them down into smaller easier to manage components, for example “create website” becomes “formulate text content for each web page”, “use html to print text content onto X number of pages” ,“use html to create links to relevant pages to improve UX by improving website navigability”,etc, this system is known as kanban.


However on the flipside it can be difficult to handle huge projects with this system, especially when there are multiple teams involved, spread across various physical locations, it makes organisation and standardisation a superfluous and frustrating process, there is also a problem with storage limits, with each card limited to the storage of 10mb for each upload for basic member, which increases for business and gold users at 250 mb.


However for our purposes, in the context of a small project, these tools work remarkably well together, utilising these applications, trello to manage time and task management, as well as the sharing of information, and visual studio with its various features as mentioned above which make the creation of projects as simple as possible help me conclude that the choice of these IT tools were appropriate for my project.


If this was a team project, it would probably be better to delegate roles, such as one person is responsible for dealing with the content of the website, another would print the content within the html, and another person would deal with how the website should look, as well as researching if such a design is pragmatic with our current skill level, and another person would be responsible for the CSS aspect of the site once the HTML is done. This would have improved productivity, however it would be at the cost of efficiency, as output would be significantly slower.


This could be combated by, for example, time management, tracking and limiting how much time is spent on each task, taking regular breaks, attending and holding stand up meetings only when necessary, forsaking multi tasking, and other similar work optimisations.
